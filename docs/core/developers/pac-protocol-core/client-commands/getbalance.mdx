# getbalance
## Command

```bash
pacprotocol-cli getbalance ( "account" minconf addlocked include_watchonly )
```

If account is not specified, returns the server's total available balance.
If account is specified (DEPRECATED), returns the balance in the account.
Note that the account "" is not the same as leaving the parameter out.
The server total may be different to the balance in the default "" account.



## Arguments

### `account` 
*(string, optional)*

DEPRECATED. The selected account, or "*" for entire wallet. It may be the default account using "".


### `minconf` 
*(numeric, optional, default=1)*

Only include transactions confirmed at least this many times.


### `addlocked` 
*(bool, optional, default=false)*

Whether to include the value of transactions locked via InstaPAC in the wallet's balance.


### `include_watchonly` 
*( bool, optional, default=false)*

Also include balance in watch-only addresses (see 'importaddress')


## Examples


The total amount in the wallet with 1 or more confirmations

```bash
pacglobal-cli getbalance 
```



The total amount in the wallet at least 6 blocks confirmed

```bash
pacglobal-cli getbalance "*" 6
```



As a json rpc call

```bash
curl --user myusername --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "getbalance", "params": ["*", 6] }' -H 'content-type: text/plain;' http://127.0.0.1:1111/
```
## Result
```bash
amount              #(numeric) The total amount in PAC received for this account.
```
